# 門市排班系統 — Product Specification v1.0

## 1. 產品概述

一個**單人使用、純前端的網頁應用**，幫助門市排班管理者自動產生每月班表。

使用者設定員工清單、特別日、個人限制後，系統透過**優先序規則引擎**自動產生最佳班表，並支援人工調整與 Excel 匯出。

- 無需後端伺服器
- 所有資料儲存於瀏覽器 `localStorage` / `IndexedDB`
- 語言：繁體中文
- 單一使用者，無帳號登入系統

## 2. 術語定義

| 術語 | 定義 |
|------|------|
| `F05` | 早班 (08:00–18:00) |
| `F13` | 晚班，一般平日 (12:00–21:30) |
| `A` | 中班，等同晚班，用於店務日 (10:00–21:30) |
| `休` | 休假（非例假的一般休假）|
| `例` | 例行休假（勞基法例假，每週至少一個）|
| `國` | 國定假日休假（不上班）|
| `國05` | 國定假日上早班（加班）|
| `國13` | 國定假日上晚班（加班）|
| `國A` | 國定假日店務日上中班（加班，等同假日晚班）|
| `特` | 特殊假別（如婚喪假，排班時保留，不計入班別邏輯）|
| `公` | 公假（排班時保留，不計入班別邏輯）|
| `PT` | 部分工時人員（僅標記，不受一般班別規則約束）|
| 假日 | 國定假日（特別日類型）|
| 店務 | 開會日（特別日類型）|
| 大清 | 大清掃日（特別日類型）|
| 四周 | 四週勞基法合規節點（特別日類型，程式自動計算）|
| 平日 | 非假日（國定假日）的一般工作日 |
| 前一個月 | 上個月最後一天的班別，用於跨月連班規則判斷 |
| 四周結轉 | 當月第一個四周節點的稽核週期中，在上個月已累積的「例」與「休」天數；每月排班的必要輸入 |

## 3. 資料模型

### 3.1 Employee（員工）

```typescript
{
  id: string,            // UUID
  name: string,          // 員工姓名（顯示用）
  isSupervisor: boolean, // 是否為主管
  isVeteran: boolean,    // 是否為老手（老手與主管可同時為 true）
  isPT: boolean,         // 是否為部分工時（PT 員工不受一般規則約束）
  isActive: boolean,     // 是否啟用（停用的員工不排入班表）
  prevMonthLastShift: ShiftType | null  // 上個月最後一天的班別，每月開始排班前更新
}
```

### 3.2 ShiftType（班別列舉）

```typescript
type ShiftType =
  | "F05"   // 早班
  | "F13"   // 晚班
  | "A"     // 中班（等同晚班）
  | "休"    // 休假
  | "例"    // 例行休假
  | "國"    // 國定假日休假
  | "國05"  // 國定假日早班
  | "國13"  // 國定假日晚班
  | "國A"   // 國定假日店務日中班
  | "特"    // 特殊假（不受規則約束，排班前手動填入）
  | "公"    // 公假（不受規則約束，排班前手動填入）
  | "PT"    // 部分工時（僅 PT 員工適用）
```

### 3.3 SpecialDayType（特別日類型）

```typescript
type SpecialDayType = "假日" | "店務" | "大清" | "四周"
// 四周 由程式自動計算；其餘三種由使用者手動標記
// 一天可同時為「四周」加一或多種手動類型
// 「假日」可與「店務」或「大清」疊加；「店務」與「大清」互斥
```

### 3.4 SpecialDay

```typescript
{
  date: string,           // "YYYY-MM-DD"
  type: SpecialDayType
}
```

### 3.5 PersonalConstraint（月度個人限制）

```typescript
{
  employeeId: string,
  month: string,            // "YYYY-MM"
  forcedDaysOff: string[]   // 強制休假日，格式 "YYYY-MM-DD"
}
```

### 3.6 ScheduleEntry（班表單格）

```typescript
{
  employeeId: string,
  date: string,            // "YYYY-MM-DD"
  shift: ShiftType,
  isAutoRelaxed: boolean,  // 因規則被放寬而排定（UI 顯示淺黃色）
  isManualEdit: boolean    // 使用者手動修改（UI 顯示淺藍色）
}
```

### 3.7 CycleCarryIn（四周結轉）

記錄每位員工在當月**第一個四周稽核週期**中，於**上個月**已累積的例假與休假天數。

```typescript
{
  employeeId: string,
  reiCount: number,  // 從 prevFourWeekDate + 1 日到上個月月底，累積的「例」天數
  xiuCount: number   // 從 prevFourWeekDate + 1 日到上個月月底，累積的「休」天數
}
```

**說明：**
- 若第一個四周週期的開始日（`prevFourWeekDate + 1`）在當月第一天或之後，則該週期不跨月，所有員工的 `reiCount` 與 `xiuCount` 皆為 `0`，不需輸入
- 若第一個四周週期跨越上個月（即 `prevFourWeekDate + 1` 早於當月第一天），則需輸入結轉值
- 當月的**第二個四周節點**（若有）其週期必定完全在當月之內，不需結轉
- 系統可從上個月的班表自動計算；若上個月班表不存在，則需使用者手動輸入

### 3.8 MonthlySchedule（月班表）

```typescript
{
  month: string,                    // "YYYY-MM"
  prevFourWeekDate: string,         // 使用者輸入的上次四周節點日期 "YYYY-MM-DD"
  cycleCarryIn: CycleCarryIn[],     // 每位員工的四周結轉（見 3.7）
  specialDays: SpecialDay[],        // 含自動計算的四周節點
  constraints: PersonalConstraint[],
  entries: ScheduleEntry[],
  relaxedRules: {
    ruleId: string,
    ruleName: string,
    affectedDates: string[]         // 被影響的日期
  }[]
}
```

## 4. 規則系統

### 4.1 設計原則

- 所有業務規則皆有 **priority**（整數，越小越重要）
- 排班時，系統先嘗試滿足所有規則
- 若排不出班表，移除 priority 數值最大（最不重要）的規則後重試
- 重複此流程直到排出班表，或所有規則皆被移除
- 最終報告哪些規則被移除、影響哪些日期
- 使用者可在「規則設定」頁調整 priority 順序

### 4.2 物理限制（永遠生效，不可被移除）

| 限制 | 說明 |
|------|------|
| 單人單班 | 每人每天只能有一個班別 |
| 班表完整 | 每位啟用員工每天都必須被分配一個班別（或假別）|
| `特`、`公` 鎖定 | 排班前手動填入的特、公假不可被程式覆蓋 |
| PT 員工豁免 | PT 員工不受一般排班規則約束，班別由使用者手動填入 |

### 4.3 規則清單（預設 priority 排序）

| Priority | Rule ID | 規則名稱 | 詳細說明 |
|----------|---------|---------|---------|
| 1 | R01 | 指定休假 | 每人設定的強制休假日必須排入（班別為「休」）|
| 2 | R02 | 四周合規 | 每個四周週期（28 天，從上次四周節點翌日起算）內，每位啟用員工至少有 4 個「例」與 4 個「休」。**第一個節點的稽核需加入 `cycleCarryIn` 的結轉天數**（詳見第 5 節）|
| 3 | R03 | 主管班別覆蓋 | 每天必須有一名主管排早班（F05 或 國05），且有一名主管排晚班（F13、A、國13 或 國A）|
| 4 | R04 | 主管不兼任 | R03 中的早班主管與晚班主管不得為同一人 |
| 5 | R05 | 主管不同天全休 | 同一天不得所有主管都休假（休、例、國任一）|
| 6 | R06 | 老手早班保障 | 每天早班（F05 或 國05）至少有一名老手 |
| 7 | R07 | 老手不同天全休 | 同一天不得所有老手都休假（休、例、國任一）|
| 8 | R08 | 平日人力 | 平日每天恰好 3 人排 F05、恰好 4 人排 F13 |
| 9 | R09 | 晚班隔天禁排早班 | 若某員工前一天班別為 F13、A、國13 或 國A，則隔天不得排 F05 或 國05（含跨月，使用 `prevMonthLastShift` 判斷第一天）|
| 10 | R10 | 假日人力 | 假日每天恰好 3 人排 國05、恰好 5 人排 國13（若假日同為店務日則晚班改為 國A）|
| 11 | R11 | 店務日班別轉換 | 店務日所有 F13 自動改為 A；若店務日同時為假日，則 國13 改為 國A |
| 12 | R12 | 國定假日「國」分配 | 假日當天，所有啟用員工（非特、非公）的班別必須帶「國」前綴（國05、國13/國A、國），由系統依人力需求自動分配 |
| 13 | R13 | 每週至少一個例假 | 每個自然週（週一至週日），每位啟用員工至少有一個「例」；跨月的週只計算當月範圍內的天數 |
| 14 | R14 | 不連續排超過五天 | 每位員工連續上班天數（F05、F13、A、國05、國13、國A）不超過 5 天，休、例、國可中斷計數 |
| 15 | R15 | 大清日人力 | 大清日至少 8 人上班，若超過 8 人需安排，多餘人員優先排晚班（F13 或 A）|

### 4.4 規則互動說明

| 情況 | 處理方式 |
|------|---------|
| 假日 + 店務 同一天 | R10 決定人力（3早5晚），R11 將晚班改為 國A |
| 假日 + 大清 同一天 | R10 決定人力（3早5晚），R15 的 8 人需求與 R10 的 8 人（3+5）需求重疊，不額外衝突 |
| 店務 + 大清 同一天 | UI 阻止此組合（互斥）|
| 某日同時為「四周」節點 | 四周節點只影響 R02 的稽核週期劃分，不影響當天班別排定 |

## 5. 四周日期計算與 R02 稽核邏輯

### 5.1 四周節點計算

```
輸入：使用者提供的「上次四周節點日期」（prevFourWeekDate）

計算邏輯：
  candidate = prevFourWeekDate + 28 天

  while candidate 在當月範圍內:
    標記 candidate 為「四周」特別日
    candidate = candidate + 28 天

注意：四周節點可與「假日」、「店務」、「大清」疊加於同一天。
```

### 5.2 R02 稽核週期與結轉邏輯

一個月最多可跨兩個四周節點，每個節點代表一個稽核週期的結束。

**各週期的稽核範圍：**

```
週期 1（第一個節點）：
  開始日 = prevFourWeekDate + 1（可能在上個月）
  結束日 = 當月第一個四周節點日
  當月天數 = 結束日的日數（例：節點在 6/12，當月範圍為 6/1–6/12）
  結轉天數 = cycleCarryIn（開始日到上月月底在上月已累積的例/休）

  R02 判斷：
    total_例 = cycleCarryIn.reiCount + 當月 6/1–6/12 排入的「例」天數
    total_休 = cycleCarryIn.xiuCount + 當月 6/1–6/12 排入的「休」天數
    合規條件：total_例 >= 4 且 total_休 >= 4

週期 2（第二個節點，若有）：
  開始日 = 當月第一個四周節點 + 1（必定在當月內）
  結束日 = 當月第二個四周節點日
  結轉天數 = 0（週期完全在當月，無需結轉）

  R02 判斷：
    合規條件：當月該範圍內的「例」>= 4 且「休」>= 4
```

### 5.3 具體範例

```
情境：
  上次四周節點 = 5月15日（prevFourWeekDate）
  當月 = 6月（30天）

自動計算的四周節點：
  5月15日 + 28 = 6月12日  → 當月第一個節點
  6月12日 + 28 = 7月10日  → 不在當月，不標記

稽核週期：
  週期 1：5月16日 → 6月12日（共 28 天）
    結轉輸入（使用者填入或上月自動計算）：
      例如：屈澤宇在 5/16–5/31 共有 例×3、休×2
      在 6/1–6/12 還需至少 例×1、休×2
    R02 判斷時：
      total_例 = 3（結轉） + 6月份的例 >= 4
      total_休 = 2（結轉） + 6月份的休 >= 4

  週期 2：無（7/10 不在當月）
```

### 5.4 cycleCarryIn 的來源

| 情況 | 來源 |
|------|------|
| 上個月已在本系統排班 | 系統自動從上月班表計算，並在 Step 1 自動填入 |
| 首次使用本系統（無上月資料）| 使用者在 Step 1 手動輸入每人的結轉天數（預設為 0）|
| 新進員工（本月才加入）| 該員工結轉值固定為 0 |

## 6. 排班演算法（High-Level）

```
輸入：
  employees[]        啟用中的員工（含角色、prevMonthLastShift）
  daysInMonth        當月天數
  specialDays[]      特別日清單（含四周節點）
  constraints[]      個人限制（強制休假）
  rules[]            規則清單（依 priority 升序排列）
  lockedEntries[]    已鎖定格子（特、公假）

演算法：

  1. Pre-fill（前置填入）
     a. 將所有「特」、「公」填入對應格子（鎖定，後續不可覆蓋）
     b. 將所有「指定休假」(R01) 填入

  2. Constraint Relaxation Loop
     activeRules = 所有規則（依 priority 升序）
     relaxedRules = []

     loop:
       result = attemptSchedule(employees, days, specialDays,
                                activeRules, lockedEntries)
       if result.success:
         return { schedule: result.schedule, relaxedRules }
       else:
         if activeRules is empty:
           return FAILURE  →  顯示錯誤原因
         droppedRule = activeRules.pop()   // 移除 priority 最大的
         relaxedRules.push({
           rule: droppedRule,
           affectedDates: result.conflictDates
         })
         continue

  3. attemptSchedule 內部
     - 使用 Backtracking + Constraint Propagation
     - 按日期順序逐日排班
     - 對每天的每位員工嘗試所有合法班別組合
     - 若某分配導致後續日期無解，回溯並嘗試下一組合
     - 返回 { success: bool, schedule?, conflictDates? }
```

### 7.4 月度排班頁（Stepper 導引，共 5 步）

#### Step 1：月份設定

**基本設定：**
- 年月選擇器（YYYY-MM）
- 輸入「上次四周節點日期」（日期選擇器）
- 系統自動計算並顯示本月四周節點日期

**四周結轉輸入（自動或手動）：**

系統判斷上個月班表是否已儲存於本應用：

- ✅ **有上月資料**：自動計算每位員工的 `cycleCarryIn`，顯示預覽表格，使用者確認後帶入
- ❌ **無上月資料（首次使用或缺漏）**：顯示手動輸入表格

手動輸入表格格式：

| 員工姓名 | 上月已累積「例」天數 | 上月已累積「休」天數 |
|---------|-------------------|-------------------|
| 屈澤宇  | [輸入框，預設 0]  | [輸入框，預設 0]  |
| 楊琇涵  | [輸入框，預設 0]  | [輸入框，預設 0]  |
| …       | …                 | …                 |

說明文字：「請填入每人從上次四周節點（{prevFourWeekDate}）翌日起，到上個月月底為止，累積的例假和休假天數。若為首月使用或不確定，可填 0（系統會盡量在本月補足）。」

若第一個四周週期的開始日（`prevFourWeekDate + 1`）在當月第一天或之後，則無需輸入結轉，系統自動隱藏此區塊。

#### Step 2：特別日標記

- 月曆格式顯示
- 四周節點：自動標記（唯讀，灰色底）
- 其他日期：點選後出現選單，可切換假日標記，並在店務 / 大清 / 清除店務大清之間選擇

色碼：

| 類型 | 顏色 |
|------|------|
| 假日 | 紅色底 |
| 店務 | 橘色底 |
| 大清 | 藍色底 |
| 四周 | 灰色底（可疊加）|

#### Step 3：個人限制輸入

- 表格格式（列 = 員工，欄 = 日期）
- 點選格子標記「強制休假」，再點取消
- 最右側顯示每人已設定的強制休假天數

#### Step 4：產生班表

- 大型「產生班表」按鈕
- 產生中顯示 loading 動畫
- 結果顯示：
  - ✅ 成功：所有規則均滿足，進入 Step 5
  - ⚠️ 部分放寬：列出被移除的規則名稱 + 受影響日期，提供「前往查看班表」按鈕
  - ❌ 失敗：顯示原因（例：人力不足），引導使用者修改設定

#### Step 5：檢視 / 調整 / 匯出

**班表表格**（列 = 員工，欄 = 日期）：

```
欄首列 1：特別日標記（假日 / 店務 / 大清 / 四周）
欄首列 2：日期數字（1, 2, 3 … 31）
欄首列 3：星期（日/一/二/三/四/五/六）
第 1 資料欄：「前一個月」（顯示 prevMonthLastShift，唯讀）

```

## 8. 匯出規格（Excel）

### 8.1 參考格式

以使用者提供的 `6月班表有國.xlsx`（工作表：`排班_空白`）為版面範本，精確重現其欄位結構。

### 8.2 欄位結構

```
欄 A：員工姓名
欄 B：員工角色代號（G = 主管, J = 老手, ⅹ = 其他，依原始慣例）
欄 C：「前一個月」（上個月最後一天的班別）
欄 D 起：當月每一天的班別（最多 31 欄）
最後幾欄：統計欄
  - 未排天數
  - 實際休假天數
  - 預計休假天數
  - 4周內休假天數
  - 例（例假計數）
  - 休（休假計數）
  - 指（指定休假計數）
```

### 8.3 列結構

```
第 1 列：特別日標記列（假日/店務/大清/四周 標記在對應日期欄位）
第 2 列：日期（Excel 日期序號，欄 C 顯示「前一個月」文字）
第 3 列：星期（日/一/二/三/四/五/六）
第 4 列起：每位員工一列
底部統計列：
  - F05 班人數（每日）
  - F13 班人數（每日）
  - A 班人數（每日，含 國A）
  - 總人數（每日）
  - 排班結果（格式：xF05 0F01 xF13 xA）
  - 上班需要人力
  - 檢定結果（A = 合格, B = 不合格）
```

### 8.4 工作表命名

主班表工作表命名為：`排班_YYYY年M月`（例：`排班_2026年6月`）

## 9. Edge Cases

| 情況 | 處理方式 |
|------|---------|
| 某月無假日 | R10、R12 自動跳過（不納入 relaxation loop）|
| 某月無大清日 | R15 自動跳過 |
| 某月無店務日 | R11 自動跳過 |
| 某月有多天假日 | 每天各自套用 R10、R12；每人同月可能有多個「國」班 |
| 主管人數不足（< 2）| R03、R04 邏輯上無法同時滿足；排班前預先偵測並在 UI 顯示警告 |
| 啟用員工數少於每日最低需求 | 排班前預先計算並顯示錯誤，阻止產生班表 |
| 跨月例假週 | R13 只計算當月範圍內的天數 |
| 跨月連班判斷 | R09 使用 `prevMonthLastShift` 判斷當月第一天 |
| 第一個四周週期不跨月 | `cycleCarryIn` 全員為 0，Step 1 自動隱藏結轉輸入區塊 |
| 首次使用系統（無上月資料）| 使用者手動輸入結轉值，或填 0 讓程式盡量在本月補足 |
| 當月只有一個四周節點 | 只有週期 1 需要結轉；無週期 2 |
| 當月有兩個四周節點 | 週期 1 需結轉，週期 2 完全在當月，結轉為 0 |
| 新進員工（本月才加入）| 該員工 `cycleCarryIn` 固定為 0 |
| `特`、`公` 與規則衝突 | 鎖定格子不計入人力統計，不受任何規則影響 |
| 員工同時為主管 + 老手 | R03–R05 與 R06–R07 均適用 |
| 同一天標記多種特別日 | UI 阻止店務 + 大清；假日與四周可疊加於店務或大清 |
| 所有規則移除後仍無解 | 顯示錯誤訊息說明原因，引導使用者修改設定 |

## Appendix A：資料持久化清單

| 資料 | 存放位置 | 清除時機 |
|------|---------|---------|
| 員工清單 | `localStorage` | 使用者手動刪除 |
| 規則優先序設定 | `localStorage` | 使用者手動還原預設 |
| 月度班表（含 cycleCarryIn） | `IndexedDB` | 使用者選擇清除或切換月份 |
| 個人限制 | 附屬於月度班表 | 隨月度班表清除 |

**跨月資料流：**

```
本月班表儲存時，系統同時計算並儲存：
  - 每位員工本月最後一天的班別（→ 下月的 prevMonthLastShift）
  - 每位員工在本月最後一個四周節點後，到本月月底累積的例/休天數
    （→ 下月的 cycleCarryIn，若下月第一個四周節點跨越本月）

下個月建立排班時，系統自動帶入以上兩筆資料。
```
